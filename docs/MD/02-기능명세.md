# 기능 명세

현재 구현된 기능을 사용자 흐름과 내부 규칙 중심으로 정리한다.
상태: ✅ 구현·검증 / 🟡 부분(스캐폴드) / ⛔ 미구현.

---

## 1. 무기 검색 (상단 헤더) ✅
- 검색창은 **상단 헤더 중앙**(탭이 있던 자리)에 위치. 입력 즉시 **플로팅 자동완성 드롭다운**으로 결과 표시
  — 아래 필터 UI 를 밀어내지 않고 위를 덮으며, 무기 선택/외부 클릭/ESC 로 닫힘.
- **카테고리 필터(헤더 "⚙ 필터" 팝오버)**: 속성·종류·등급·슬롯·탄약·**프레임(아키타입)**·**기원 특성**·**시즌(복각)**
  칩 + 전체 15스탯 min/max + 퍽 보유/제외 + **DIM식 텍스트 쿼리**.
  - 같은 카테고리 내 OR, 카테고리 간 AND.
  - **컨텍스트 인지 패싯**: 현재 검색/필터 결과에 맞춰 각 카테고리가 해당하는 값만, 그 결과 기준 갯수로
    표시(예: '폭발 용광로' → 속성 물리(3), 종류 파동 소총(3)). 각 카테고리는 자기 필터를 제외하고 계산해
    OR 다중선택을 유지.
  - 필터 옆에 **현재 매칭 무기 수** 배지 표시(`N개`), 드롭다운 상단에 `N개 무기 · 상위 60개 표시`.
  - 카테고리 **접기/펼치기**(상태 localStorage 저장) + **배치(순서) 변경** 설정 패널(⚙ 배치, localStorage 저장).
- API: `GET /weapons`(다중값 CSV + query + 스탯), `GET /filters`(컨텍스트 패싯), `GET /weapons/count`,
  `GET /search/help`(쿼리 치트시트), `GET /perks?q=`.

### DIM식 텍스트 쿼리 언어 ✅ (확장성)
- `is:<종류/속성/슬롯/등급/탄약/adept/holofoil/exotic/legendary/randomroll>` (EN+KO 토큰),
  `perkname:"이름"`/`perk:`, `stat:<키>:<>=|<=|>|<|=|a-b>`, `season:<조건>`, `frame:`, `origin:`,
  `name:`/맨텍스트, 불리언 `and`(암묵)·`or`·`not`/`-`·괄호 `()`.
- 예) `is:핸드캐논 stat:range:>=50 -perkname:"무법자" season:>=23`.
- 구현: `backend/app/query.py`(토크나이저+파서+SQL 컴파일러+**필터 레지스트리** — 새 필터 1줄 추가).
  값은 항상 파라미터 바인딩, 키워드/스탯키/연산자는 화이트리스트(인젝션 차단), 문법 오류는 400.

## 2. 변형·복각·홀로포일·시즌 처리 ✅ (정확성 핵심)
- DIM 은 item hash 로 매칭되지만, 같은 무기가 여러 hash 로 존재: **홀로포일**(`isHolofoil`, 외형만 다름·
  퍽 풀 동일), **시즌별 복각**(이름·종류 같고 퍽 풀 다름), **에이뎁트**(`isAdept`). 모두 매니페스트 플래그로 식별.
- 검색은 **시즌(=변형그룹+워터마크) 단위**로 1행씩 접어 표시. 같은 시즌 홀로포일은 한 행으로 병합,
  시즌이 다르면 같은 이름이라도 별도 행. 각 행에 **`S{번호} · 시즌명`**(워터마크→시즌 매핑) + 워터마크 아이콘.
- 더미(`itemType≠3`)·트래커/코스메틱 플러그는 적재 단계에서 제외.
- **위시리스트 내보내기 시 변형 자동 전개**: 한 무기에 만든 롤을 같은 변형 그룹의 모든 hash 에 emit
  (변형별 퍽 풀로 필터 — 못 굴리는 조합은 생략). 한 줄의 위시리스트가 홀로포일·복각 전체를 커버.
- 데이터: `weapons.frame/frame_hash/is_holofoil/is_adept/is_featured/variant_group`,
  watermark→시즌 매핑 `backend/app/seasons.py` + `backend/app/refdata/{watermark-to-season,season-names}.json`.

## 3. 비주얼 퍽 선택기 ✅ (핵심 가치)
- 선택 무기의 **열(컬럼)별 퍽 풀** 그리드(총열/탄창/특성/기원). 선택 시 앰버 링+글로우.
- **퍽 호버 툴팁**: 한국어 설명 + 스탯 델타 칩.
- **실시간 스탯 패널**: 선택한 퍽의 스탯 델타를 합산해 무기 스탯 막대에 ▲/▼ 로 즉시 반영(d2gunsmith 식).
- **인기도 막대**, 배지(`큐레이티드`·`단종`).
- API: `GET /weapons/{hash}` (열·퍽·설명·스탯델타·인기도·변형/시즌 정보 포함).

## 4. 다중 퍽 OR → 다중 줄 자동 전개 ✅
- 한 열에서 여러 퍽 = OR. 컴파일 시 **열 간 카르테시안 곱**으로 줄 전개(줄 내부는 AND).
- DoS 방지로 한 롤의 조합 수 상한(2000) — 초과 시 400.

## 5. 트래시 롤 / 와일드카드 / 태그 / 메모 ✅
- 트래시 롤(음수 해시), 와일드카드(`-69420`, desirable 전용), 태그 칩(`[PvP]` baking), 메모(`#notes:`, `|` 제거).

## 6. 위시리스트 관리 + 가져오기 ✅
- 좌측 "내 리스트"에 롤 누적(추가/삭제/전체 비우기), 제목/설명.
- **외부 위시리스트 가져오기(⤓)**: 다른 사람이 만든 DIM `.txt` 업로드 → 롤 일괄 로드.
  - 전체 파일 파싱(`title`/`description`, `// 무기명`, `//notes:` 블록, 멀티라인).
  - 같은 무기 블록의 여러 줄(카르테시안 전개)을 **1개 롤로 역병합**(퍽 합집합) + 각 퍽을 실제 열에 매핑.
  - DB 에 없는 무기/와일드카드는 건너뛰고 요약(가져옴 N · 제외 내역) 표시.
  - 검증: MrCharles Aegis 위시리스트(약 15k 줄) → 713개 롤 가져옴.
- API: `POST /import-wishlist`(text → 롤 목록·제목·설명·집계).

## 7. 실시간 미리보기 & 내보내기 ✅
- 완성 위시리스트 텍스트 실시간 미리보기, `.txt` 다운로드. 시드 모드 경고.
- API: `POST /compile`(단일 롤), `POST /export`(완성 파일). 둘 다 조합 폭발 시 400.
- 포맷 규칙(컴파일러 강제): 쉼표=AND, OR→다중줄, 강화 퍽 미출력, 노트 `|` 금지, LF/BOM 없음, `// 무기명` 주석.

## 8. 점수화 — 위시리스트 기반·퍽 롤 중심 ✅ (재설계)
- **점수는 선택한 퍽 롤로 결정**: 점수 = 선택 퍽들의 선호도 합(desirable +, 트래시 −) + 매칭 조합 가점.
  **스탯 가중치는 점수에 미반영**(표시용으로만 계산).
- **기준은 위시리스트**(내 리스트 또는 가져온 위시리스트): desirable 롤에 든 퍽 = 가점, 트래시 = 감점,
  조합은 공동출현으로 학습. **컨텍스트 인지**(같은 퍽도 무기 종류/프레임/개별 무기에 따라 다름).
- **스코프 비중(블렌드)**: 퍽 가중치 = **동일 무기(0.60)·동일 프레임(0.25)·동일 종류(0.15)** 위시리스트를
  각각 계산한 **절대 가중합**(없는 스코프=0). 같은 퍽(=같은 plug_hash)은 무기 간 공유되므로, 무기별 데이터가
  없어도 같은 종류·프레임 위시리스트에서 평가가 **전이**된다.
- **열 비중(세분화)**: 총열/탄창/특성/기원 열의 점수 기여를 차등 — **특성 1.0 · 총열·탄창 0.35 · 기원 0.2 ·
  intrinsic 0**. 실제 롤은 열당 1개이므로 **열별 최선 퍽**만 채택.
- **동적 만점(기대치 환산)**: 점수 = 100 × (열가중 기여 + 조합) / **무기별 만점**(채점가능 열 비중 합).
  → "이상적 풀롤 대비 %"가 되어 일반화가 줄고 천장이 올라간다(예: 특성 2개만 ≈ 74, 풀롤 = 100).
- **미등록 무기 캡(coverage)**: 무기별 위시리스트가 없으면 무기 비중(0.60)이 비어 **coverage(종류+프레임=0.40)
  로 점수가 캡**된다 → 새 무기를 종류/프레임 기준으로 부분 평가하되 낮은 신뢰가 점수에 반영. 빌더에
  "종류·프레임 기준 — 최대 40%" 안내. 신호가 전혀 없으면 **점수 없음(None)**.
- 분류 임계: 기본 god≥75, viable≥40(동적 만점 재보정, 프로필에서 조정). 빌더에 실시간 점수 배지(퍽/조합 breakdown).
- **퍽별 점수 표시**: 빌더 퍽 그리드에서 각 퍽 타일에 **그 퍽의 선호도 점수**(가중치×50, desirability)를
  배지로 표시하고 **점수에 따라 색을 구분**한다 — 핵심(+35↑) 앰버 · 좋음(+15↑) 초록 · 약한 선호(+) 파랑 ·
  트래시(−) 빨강. 좌측 액센트/옅은 배경 틴트로 한눈에 구분. 툴팁엔 **이 무기 점수 기여**(열 비중 환산)도 표시
  → 같은 선호도라도 특성 > 총열/탄창 기여 차이가 보인다. 활성 프로필 + 해당 무기/종류/프레임 신호가 있을 때만 표시.
- 프로필(JSON=공유 단위): 스코프 비중·열 비중·컨텍스트 가중치/조합 + 임계값. "점수 기준" 탭에서
  위시리스트(또는 `.txt`)로 학습·저장·활성화·JSON 내보내기/가져오기. 비중은 기본값 적용 + JSON 으로 조정.
- API: `GET/POST/DELETE /scoring-profiles`, `POST /score`(점수·분류·breakdown·coverage·max_possible),
  `POST /score/perk-weights`(퍽별 가중치 맵·has_signal·scale·max_possible·coverage·column_weights·kinds),
  `POST /scoring/derive-weights`.

## 11. UI / 레이아웃 ✅
- **우측 드로어**: 헤더 탭(빌더/점수기준/내창고/메타)+상태를 햄버거(☰→X 모핑) 드로어로 통합(오프캔버스 슬라이드,
  딤 스크림 오버레이).
- **빌더 3분할**: 좌측 레일(내 리스트) · 중앙(무기 헤더 + 퍽 그리드) · **우측 "현재 롤" 패널**.
  롤 옵션(트래시 롤/와일드카드/태그/메모)과 **위시리스트 추가** 버튼을 우측 패널로 이동, 그 아래에 리스트
  컴파일/내보내기. **합계 점수(/100·등급)는 우측 패널 최상단에 크게** 표시되고 패널은 **sticky** 라 퍽
  그리드를 스크롤해도 점수·추가 버튼이 항상 보인다(좁은 화면 ≤1180px 에선 세로 스택).
- 데이터 소스 상태 칩(샘플/매니페스트 + 무기 수).

## 12. 보안 ✅
- 단일 사용자 로컬 도구 전제. 도커 published 포트는 **127.0.0.1 바인딩**(LAN 미노출).
- SQLi/XSS/경로순회/비밀노출 없음, 입력 DoS 상한(import 8MB·조합 2000), CORS 명시적 허용목록,
  OAuth state TTL/상한. 자동 검증: `python scripts/security_check.py`(21개 항목). → [05-보안검증.md](05-보안검증.md).
