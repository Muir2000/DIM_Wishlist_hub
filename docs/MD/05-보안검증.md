# 보안 검증

## 위협 모델
**단일 사용자 로컬 도구**. 자기 PC 에서 브라우저로 접속해 위시리스트를 만들고 Bungie 계정(선택)으로
창고를 가져온다. 따라서 "포트가 네트워크에 노출될 경우"의 위험을 차단하는 데 초점을 둔다.

## 검증 절차 (반복 실행)
스택을 기동한 뒤:
```powershell
python scripts/security_check.py                       # 기본 http://127.0.0.1:8080/api (nginx)
$env:SECCHK_BASE="http://127.0.0.1:8000"; python scripts/security_check.py   # 백엔드 직접
```
정적 점검 + 런타임 프로빙 **21개 항목**을 수행하고, 하나라도 실패하면 비0 종료한다.
**현재 결과: 21/21 통과.**

### 점검 항목
**정적(설정/소스)**
- 도커 published 포트가 모두 `127.0.0.1` 바인딩(LAN 미노출).
- `.env` 가 `.gitignore`·`.dockerignore` 에서 제외(비밀/토큰 DB 미커밋·미이미지화).
- CORS `allow_origins` 가 와일드카드(`*`) 아님(credentials 동반 안전).
- nginx `client_max_body_size` 설정(업로드 한도).

**런타임(실행 중 앱)**
- `/health` 200, `/status` 가 API 키/토큰/시크릿을 노출하지 않음(bool 만).
- **SQLi**: 이름 검색(`q`)·텍스트 쿼리(`query`)에 인젝션 페이로드 → 200/400, 500·트레이스백 없음, weapons 테이블 무결.
- **DoS**: `/compile` 조합 폭발 → 400, `/import-wishlist` 대용량(>8MB) → 413/422, 정상 import 는 200.
- **에러 위생**: 잘못된 스탯필터 → 500 아님, 잘못된 쿼리 → 400(친절 메시지), 404·OAuth 잘못된 state → 트레이스백 없음.

## 적용된 보안 조치 (감사 → 수정)
| 영역 | 조치 |
|---|---|
| SQL 인젝션 | 모든 값 `?` 파라미터 바인딩. `query.py` 는 키워드/스탯키/연산자/`is:` 토큰 화이트리스트, 값 바인딩. 컬럼명 f-string 은 내부 상수만. |
| XSS | React 자동 이스케이프. `dangerouslySetInnerHTML`/`innerHTML`/`eval` 없음. |
| 포트 노출 | docker-compose published 포트 `127.0.0.1:8000`/`127.0.0.1:8080` 바인딩(Swagger·무인증 엔드포인트 LAN 차단). |
| 입력 DoS | 한 롤 조합 상한 `MAX_ROLL_COMBOS=2000`(compile/export 400), import `text` 최대 8MB, nginx 16MB. |
| OAuth | state `{state: 만료}` TTL 10분 + 상한 256(무한 증가 차단), 검증 시 1회용 소비. |
| 에러 노출 | uvicorn `--reload`/debug 미사용(트레이스백 미노출). 잘못된 스탯필터 무시(500→정상), 쿼리 오류는 친절한 400. |
| CORS | 명시적 허용 목록(localhost:5173 + FRONTEND_URL), 메서드/헤더 최소화. |
| 비밀 | API 키/시크릿은 Bungie 요청 헤더로만 사용, 응답/에러에 미노출. `.env` git/docker 제외. |

## 알려진 한계 (위협 모델상 수용)
- **인증 없음**: 모든 엔드포인트 공개. 단일 사용자 로컬 전제이며 포트를 127.0.0.1 로 묶어 완화.
  LAN/공개 배포하려면 `/me/*`·`/scoring-profiles*` 등에 인증을 추가하고 바인딩을 변경할 것.
- **토큰 평문 저장**: OAuth access/refresh 토큰이 `data/app.sqlite` 에 평문. 로컬 단일 사용자 전제로 수용
  (DB 파일은 git/이미지에서 제외됨).
- watermark→시즌 매핑은 DIM 커뮤니티 스냅샷 — 보안과 무관(기능 정확성 항목).

## 재검증
코드 변경 후에는 `docker compose build && docker compose up -d` → `python scripts/security_check.py`
가 21/21 통과하는지 확인한다. 새 엔드포인트/입력을 추가하면 `scripts/security_check.py` 에 항목을 추가한다.
