# 보안 검증

## 위협 모델
**기본은 로컬 도구, 옵션으로 멀티유저(LAN/소규모 공유)**. 기본 배포는 포트를 `127.0.0.1` 에 묶어 로컬에서만
쓰지만, Bungie OAuth **로그인 + 세션 쿠키**로 프로필·위시리스트·창고를 **사용자(membership)별로 분리** 저장하므로
`WEB_BIND=0.0.0.0` 로 LAN 공유도 가능하다. 따라서 초점은 ① 포트 노출 시 위험 차단 + ② **세션 위·변조/사용자 간
데이터 격리** + ③ 입력/주입 위생이다.

> ⚠ 이 문서의 자동 검증(`scripts/security_check.py`, 21항목)은 **인증 도입 이전 표면**을 검사한다. 세션 쿠키·
> `/auth/*`·`/me/*` owner 격리 등 멀티유저 표면은 아직 자동 항목에 미포함 — **추가 검증 항목 작성이 필요**(아래 한계).

## 검증 절차 (반복 실행)
스택을 기동한 뒤:
```powershell
python scripts/security_check.py                       # 기본 http://127.0.0.1:8080/api (nginx)
$env:SECCHK_BASE="http://127.0.0.1:8000"; python scripts/security_check.py   # 백엔드 직접
```
정적 점검 + 런타임 프로빙 **21개 항목**을 수행하고, 하나라도 실패하면 비0 종료한다.
**현재 결과: 21/21 통과.**

### 점검 항목
**정적(설정/소스)**
- 도커 published 포트가 모두 `127.0.0.1` 바인딩(LAN 미노출).
- `.env` 가 `.gitignore`·`.dockerignore` 에서 제외(비밀/토큰 DB 미커밋·미이미지화).
- CORS `allow_origins` 가 와일드카드(`*`) 아님(credentials 동반 안전).
- nginx `client_max_body_size` 설정(업로드 한도).

**런타임(실행 중 앱)**
- `/health` 200, `/status` 가 API 키/토큰/시크릿을 노출하지 않음(bool 만).
- **SQLi**: 이름 검색(`q`)·텍스트 쿼리(`query`)에 인젝션 페이로드 → 200/400, 500·트레이스백 없음, weapons 테이블 무결.
- **DoS**: `/compile` 조합 폭발 → 400, `/import-wishlist` 대용량(>8MB) → 413/422, 정상 import 는 200.
- **에러 위생**: 잘못된 스탯필터 → 500 아님, 잘못된 쿼리 → 400(친절 메시지), 404·OAuth 잘못된 state → 트레이스백 없음.

## 적용된 보안 조치 (감사 → 수정)
| 영역 | 조치 |
|---|---|
| SQL 인젝션 | 모든 값 `?` 파라미터 바인딩. `query.py` 는 키워드/스탯키/연산자/`is:` 토큰 화이트리스트, 값 바인딩. 컬럼명 f-string 은 내부 상수만. |
| XSS | React 자동 이스케이프. `dangerouslySetInnerHTML`/`innerHTML`/`eval` 없음. |
| 포트 노출 | 백엔드 `127.0.0.1:8000` 고정(Swagger 로컬 전용). 프론트 `${WEB_BIND:-127.0.0.1}:8080` — 기본 로컬, `WEB_BIND=0.0.0.0` 시에만 LAN. |
| 세션/인증 | Bungie OAuth → **HMAC-SHA256 서명 쿠키**(`dimhub_session`, httponly·samesite=lax·30일). `SESSION_SECRET` env 우선, 없으면 임의 생성·`DATA_DIR` 영속. 서명 검증은 `hmac.compare_digest`(타이밍 안전). |
| 사용자 격리 | 프로필/상태/창고 쿼리를 `owner=membership` 로 스코프(`repo.*(…, owner)`), 타 사용자 데이터 접근 차단. 보호 엔드포인트는 `require_membership`(미로그인 401). |
| 입력 DoS | 한 롤 조합 상한 `MAX_ROLL_COMBOS=2000`(compile/export 400), import `text` 최대 8MB, nginx 16MB. |
| OAuth | state `{state: 만료}` TTL 10분 + 상한 256(무한 증가 차단), 검증 시 1회용 소비. |
| 에러 노출 | uvicorn `--reload`/debug 미사용(트레이스백 미노출). 잘못된 스탯필터 무시(500→정상), 쿼리 오류는 친절한 400. |
| CORS | 명시적 허용 목록(localhost:5173 + FRONTEND_URL), 메서드/헤더 최소화. |
| 비밀 | API 키/시크릿은 Bungie 요청 헤더로만 사용, 응답/에러에 미노출. `.env` git/docker 제외. |

## 알려진 한계 (위협 모델상 수용 / 후속 보강 대상)
- **자동 검증 미반영(인증 표면)**: `security_check.py` 는 인증 이전 표면(21항목)만 검사. 세션 위조 거부,
  사용자 간 데이터 격리(다른 owner 프로필/창고 접근 시 빈/401), `/auth`·`/me` 보호를 **자동 항목으로 추가**해야 함.
- **쿠키 Secure 미설정**: HTTP(localhost/LAN)에서 동작하도록 `Secure` 플래그 없음. 공개망 HTTPS 배포 시
  `Secure` 추가 필요. samesite=lax 로 기본 CSRF 완화하나, 상태변경 POST 의 명시적 CSRF 토큰은 없음(LAN/로컬 전제).
- **토큰 평문 저장**: OAuth access/refresh 토큰이 SQLite 에 평문(DB 파일은 git/이미지 제외). 신뢰된 호스트 전제.
- **LAN 노출 시**: `WEB_BIND=0.0.0.0` 면 같은 네트워크에서 접근 가능 — 신뢰된 망에서만, 방화벽은 로컬 서브넷
  한정 권장(`scripts/allow-lan-8080.ps1`). OAuth redirect/`FRONTEND_URL` 은 접속 호스트와 일치해야 로그인 동작.
- watermark→시즌 매핑은 DIM 커뮤니티 스냅샷 — 보안과 무관(기능 정확성 항목).

## 재검증
코드 변경 후에는 `docker compose build && docker compose up -d` → `python scripts/security_check.py`
가 21/21 통과하는지 확인한다. 새 엔드포인트/입력을 추가하면 `scripts/security_check.py` 에 항목을 추가한다.
