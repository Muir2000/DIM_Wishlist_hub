# 08. 멀티유저 · 점수화 확장

> 대상: `fix/apply-schema-owner-index` 브랜치(+ main의 멀티유저 머지). 단일 사용자 로컬 도구를
> **Bungie 로그인 멀티유저** + **창고(보유 롤) 점수화** + **프로필별/태그별 점수**로 확장한 작업을 정리한다.
> 검증 주의: 이 작업은 원격 NAS에서 빌드·배포되었고, 작성 환경에선 빌드/실행 검증을 못 해
> 정적 검증(타입·i18n 키 일치·참조 정합) 위주로 진행했다. 배포 후 동작 확인이 필요하다.

---

## 1. 개요

| 영역 | 한 줄 요약 |
|---|---|
| 멀티유저/인증 | Bungie OAuth + **HMAC 서명 세션 쿠키**로 사용자 식별, 모든 개인 데이터를 사용자별 스코프 |
| 프로필 | **프로필 = 위시리스트 롤 + 가중치**. "내 리스트"가 활성 프로필 기준, **멀티 프로필 선택/병합** |
| 창고(Vault) | 보유 무기의 **다중 퍽 롤**(컴포넌트 310) + 퍽별 점수 + 최선 롤, **instanceId 복사**(DIM `id:`) |
| 빌더 점수 | 종합 + **태그별(PvE/PvP/GM) 점수**·추천 퍽, **최적 롤** 버튼, 롤 로드·무기 필터·hash 복사 |

## 2. 멀티유저 / 인증

**식별 = HMAC 서명 세션 쿠키** (새 의존성 없음, `backend/app/session.py`)
- 쿠키 `dimhub_session = "<membership_id>.<hmac_sha256>"`. 위·변조는 서명으로 차단(membership_id는 공개값).
- `config.SESSION_SECRET`: env 우선, 없으면 `DATA_DIR/.session_secret`에 1회 생성·영속(바인드 볼륨이라 재시작/재빌드 후에도 유지). `SESSION_COOKIE_SECURE`(https 시 1).
- 의존성: `current_membership(request) → Optional[str]`, `require_membership`(없으면 401).

**OAuth 흐름** (`backend/app/routers/auth.py`)
- `bungie_callback`: 토큰 저장 + **표시명 저장** 후 `RedirectResponse`에 **세션 쿠키 set**.
- 신규 `GET /auth/me`(로그인/표시명), `POST /auth/logout`(쿠키 삭제). `access_token(conn, membership_id)`로 멤버십 인지 리프레시.

**사용자 스코프**
- `scoring_profiles.owner`(= membership_id): 목록/조회/저장/삭제 본인 것만, 쓰기는 로그인 필수, 타인 수정 403.
- `user_state`(신규 테이블): 사용자별 빌더 상태(선택 프로필/제목 등). `GET/PUT /me/state`.
- `inventory_items`/`oauth_tokens`: membership 기준. 인벤토리는 **세션 멤버십**으로 조회, 비로그인은 공용 `DEMO`.

**배포 주의(.env)**: `FRONTEND_URL`·`BUNGIE_OAUTH_REDIRECT_URI`를 외부 공개 주소로, Bungie 앱 redirect도 동일 등록.
프론트는 모든 요청에 `credentials: "include"`(쿠키 동봉) — `frontend/src/api.ts`.

## 3. 프로필 = 롤모음 + 가중치

- `ScoringProfile.rolls`(`backend/app/models.py`): 프로필 JSON에 위시리스트 롤을 함께 보관(export/import에 자동 포함).
- `frontend/src/store.tsx`: **"내 리스트" = 선택된 프로필들의 롤 합집합**(읽기). 편집(add/remove/clear)은 **primary 프로필**에 반영하고 `saveProfile`로 영속.
- **멀티 프로필 선택**(`ScoringProfileEditor`): 프로필 목록에 **체크박스**(내 리스트에 합칠 프로필) + **★ primary**(가중치/편집 대상).
- `mergeProfiles`(store): 선택 프로필을 점수용 1개로 결합 — `perk/context_weights` **union-max**, `blend/scope/column` **평균**, 시너지 **concat**, `use_wishlist` **OR**. 점수 호출은 이 `scoringProfile` 사용(편집/표시는 primary).
- **마이그레이션**: 로그인 시 프로필 없으면 레거시 `user_state.rolls`로 **"기본 프로필"** 자동 생성(또는 primary에 1회 이관). 로그인 사용자는 항상 최소 1개 프로필 보장.

## 4. 창고(Vault) 보유 롤

**동기화(`backend/app/routers/inventory.py` `me_sync`)**
- GetProfile 컴포넌트: `102,201,205,300,304,305,310`. **305=장착 퍽**, **310=ItemReusablePlugs**(소켓별 선택 가능 퍽 → 제작 무기의 **열당 다중 퍽**).
- `inventory_items.reusable_plugs`(JSON `{소켓idx: [plug...]}`)에 저장. ⚠ 다중 퍽을 보려면 **재동기화** 필요.

**보유 롤 조회(`POST /me/weapon-rolls`)**
- 선택 무기의 **variant_group**(복각/시즌 변형) 단위로 보유 인스턴스 매칭(`_variant_hashes`).
- `_perk_columns`: 열별 선택 가능 퍽(reusable 우선, 없으면 장착 단일), **장착 퍽 강조**(equipped). 열 정렬은 `column_index`.
- 퍽별 점수(`points` = `perk_weight_map` 가중치×scale), **최선 롤**(`best_score` = 열별 최고 퍽 조합 채점).
- `InventoryPerk`: `plug_hash`, **`base_hash`**(강화→기본), `name/icon`, `column_kind`, `column_index`, `equipped`, `points`.

**표시/유틸(`frontend/src/components/Builder.tsx` 보유 롤 그리드)**
- 무기 헤더에 무기 아이콘·이름, 퍽을 **열별 아이콘 칩**(등급색)으로, 현재 롤/최선(★) 점수.
- **고유 ID 복사**: 보유 시 그 사본의 **itemInstanceId**(예 `6917530147677281027`) 복사 → DIM `id:<…>`로 정확히 그 사본 검색. 미보유는 정의 `item_hash` 폴백.
- **불러오기(⬇)**: 보유 롤의 **다중 퍽 전부**를 빌더로 로드(`base_hash`로 그리드 기본형 매칭, `column_index` 없어도 열 단위로 누락 없이).

**진단**: `GET /me/debug-rolls`(로그인) — 원본 plug + `in_perks_table`/`in_weapon_pool`로 누락 원인(동기화 vs 매칭) 구분.

## 5. 빌더 점수 / 태그

- **종합 점수**(`/score`) + **퍽별 가중치**(`/score/perk-weights`)는 기존. 점수 호출은 병합 `scoringProfile` 사용.
- **태그별 점수**(신규 `POST /score/tags`, `backend/app/routers/scoring.py`): 위시리스트 롤을 **태그(PvE/PvP/GM/레이드)로 필터링** → `derive_context` → 태그별 점수 + **추천 퍽**(열별 최고, base 해시). 기존 점수 함수만 조합.
- 롤사이드 패널: 종합 점수 + **태그별 점수 칩** + 태그별 **"추천"** 자동선택.
- **최적 롤** 버튼(퍽 선택 헤더): 활성 프로필 가중치 기준 **열별 최고(양수) 퍽 자동 선택**(클라이언트, `perkW` 재사용).
- **롤 로드**: 내 리스트 롤 클릭 / 창고 보유 롤 ⬇ → 빌더에 무기+퍽 로드(`App.loadRoll` + `pending`).
- **내 리스트 필터**: 선택 무기로 필터(이름 일치=변형 흡수) + 전체/이 무기 토글. **무기 hash 복사** 버튼.

---

## 6. 데이터 모델 / 마이그레이션 (`backend/app/db.py`)

- 신규 테이블: `user_state(owner PK, json, updated_at)`.
- idempotent ALTER(구 DB 보강): `scoring_profiles.owner`, `oauth_tokens.display_name`, `inventory_items.reusable_plugs`, `weapons.frame_en`.
- `scoring_profiles.rolls`는 별도 컬럼이 아니라 기존 `json` blob에 포함(모델 필드).
- **인덱스 생성 순서 주의**: `idx_profiles_owner`는 `owner` 컬럼 ALTER **이후** 생성(레거시 DB에서 "no such column: owner" 부팅 크래시 회피 — `fix(db)`).
- 마이그레이션은 startup 시 매니페스트 DB에도 `apply_schema` 적용.

## 7. API 요약 (신규/변경)

| 메서드·경로 | 인증 | 설명 |
|---|---|---|
| `GET /auth/me` | 세션 | 로그인 여부 + 표시명 |
| `POST /auth/logout` | — | 세션 쿠키 삭제 |
| `GET /me/state` · `PUT /me/state` | 필수 | 사용자 빌더 상태(선택 프로필/제목) |
| `GET /scoring-profiles` 외 | 본인 | owner 스코프(쓰기 401/403). 프로필 JSON에 `rolls` 포함 |
| `POST /me/sync` | 필수 | 컴포넌트 305+310 수집(다중 퍽), 세션 멤버십 기준 |
| `POST /me/weapon-rolls` | 세션 | 보유 무기 인스턴스 + 다중 퍽·점수·최선 롤 |
| `GET /me/debug-rolls` | 세션 | 진단(원본 plug·풀 매칭 여부) |
| `POST /score/tags` | 세션 | 종합 + 태그별 점수·추천 퍽 |

## 8. 알려진 한계 / 후속

- **검증**: 작성 환경(WebDAV, Node/정상 Python 부재)에서 빌드·실행 불가 → 정적 검증 위주. 배포 후 확인 필수.
- **시즌 변형 풀 불일치**: 구 시즌 변형은 현재 매니페스트 풀에 일부 퍽이 없어 `column_index` null 가능 → 로드는 열 단위 폴백으로 누락 방지하나 그리드 강조가 빠질 수 있음.
- **멀티 프로필 점수**: 현재 `mergeProfiles`(가중치 결합) — 위시리스트 파생 가중치가 주이고 수동 가중치 결합은 단순화.
- **창고 탭 확장(예정)**: 빌더식 검색/필터, 다중 퍽 표시, **다중 선택 후 id 일괄 복사**, 정렬, 중복 탐지 등 — 별도 계획으로 진행.
- **재동기화 필요**: `reusable_plugs`(다중 퍽), 신규 매니페스트 필드는 `docker compose run --rm ingest` / `/me/sync` 후 반영.

> 배포: `git pull && sudo docker compose up -d --build backend frontend` → 강력 새로고침.
> 안정화되면 `fix/apply-schema-owner-index` → `main` 병합 권장.
